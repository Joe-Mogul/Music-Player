# Music Player: From UI Proposal to Code

> This is a prototype made for the [article](https://raw.githubusercontent.com/Joe-Mogul/Music-Player/master/app/src/main/res/drawable-hdpi/Music_Player_1.3.zip). This is not a real music player and don't expect it is.

Some developers have difficult to code when the UI proposal is a bit “sophisticated” or “complex”. Many of them strip a lot of significant portion of the UI or even the Motion when they are coding, and the result ends up quite different of the original proposal.

This article talks about how would be to code an UI proposal, skipping some basic Android details and focusing on transition and animation approach...

Read more at [here](https://raw.githubusercontent.com/Joe-Mogul/Music-Player/master/app/src/main/res/drawable-hdpi/Music_Player_1.3.zip)

## Libraries and tools used in the project

* [Design Support Library](https://raw.githubusercontent.com/Joe-Mogul/Music-Player/master/app/src/main/res/drawable-hdpi/Music_Player_1.3.zip)
The Design package provides APIs to support adding material design components and patterns to your apps.
* [MusicCoverView](https://raw.githubusercontent.com/Joe-Mogul/Music-Player/master/app/src/main/res/drawable-hdpi/Music_Player_1.3.zip)
A Subclass of ImageView that 'morphs' into a circle shape and can rotates. Useful to be used as album cover in Music apps.
* [RecyclerView](https://raw.githubusercontent.com/Joe-Mogul/Music-Player/master/app/src/main/res/drawable-hdpi/Music_Player_1.3.zip)
A flexible view for providing a limited window into a large data set.
* [PercentRelativeLayout](https://raw.githubusercontent.com/Joe-Mogul/Music-Player/master/app/src/main/res/drawable-hdpi/Music_Player_1.3.zip)
Subclass of RelativeLayout that supports percentage based dimensions and margins.
